# Correção 3 bugs da aba FU — Plano de Implementação

> **Para o Hermes:** Implementação direta, sem subagentes. Tasks sequenciais.

**Objetivo:** Corrigir 3 bugs na aba FU: (1) F-U na aba Análises ainda clonando ao iniciar nova competência, (2) F-U na aba FU não clonando ao iniciar nova competência, (3) checkbox de Conversão na aba FU sem efeito.

**Arquivo:** `C:\Users\Prime04D\crm-uaicred\index.html` (~3964 linhas)

## Diagnóstico

### Bug 1: Analises "Nova competência" ainda clona F-U

**Causa provável:** `iniciarNovaCompetenciaAnalises()` filtra `clientes` local sem recarregar dados do banco antes. Se o array local estiver desatualizado, clientes com F-U podem passar no filtro.

**Solução:** Adicionar `await carregarDados()` no início da função. Atualizar label do botão para não mencionar "follow-ups".

### Bug 2: FU "Nova competência" não clona

**Causa provável:** Mesmo problema — `iniciarNovaCompetenciaFu()` não recarrega dados antes de filtrar. Além disso, o `_dupLock` é compartilhado globalmente entre ambas as funções de competência.

**Solução:** Adicionar `await carregarDados()` no início. Usar lock separado (`_dupLockFu`) para a aba FU.

### Bug 3: Conv. na FU sem efeito

**Causa provável:** `converterClienteFu()` pode falhar silenciosamente — `sb.auth.getUser()` pode retornar erro, ou o insert pode falhar por campos ausentes. Sem `try/catch`, o erro some.

**Solução:** Adicionar `try/catch` com `console.error` e toast de erro.

## Task 2: Corrigir `iniciarNovaCompetenciaFu()` — recarregar dados + lock próprio

**Objetivo:** Garantir que a clonagem mensal na aba FU funciona. Lock separado para evitar conflito com a aba Análises.

**Local:** Função `iniciarNovaCompetenciaFu()` (~linha 3633).

### 2a: Declarar lock separado

Adicionar após `let _dupLock = false;` (~linha 2854):

```javascript
let _dupLockFu = false;
```

### 2b: Reescrever a função

```javascript
// ANTES:
async function iniciarNovaCompetenciaFu() {
  if (_dupLock) return;
  _dupLock = true;
  const [ano, mes] = (mesSelecionadoFu || new Date().toISOString().slice(0,7)).split('-').map(Number);
  const dataSeguinte = new Date(ano, mes, 1);
  const mesAlvo = dataSeguinte.toISOString().slice(0,7);
  const origem = clientes.filter(c => c.follow_up === true && c.mes_referencia === mesSelecionadoFu && c.status !== 'emitido' && c.convertido !== true && c.convertido_fu !== true);
  const existentesNoAlvo = clientes.filter(c => c.mes_referencia === mesAlvo).map(c => c.cliente);
  const todos = origem.filter(c => !existentesNoAlvo.includes(c.cliente));
  if (todos.length === 0) { mostrarToast('Nenhum follow-up para duplicar.'); _dupLock = false; return; }
  await duplicarClientesFu(todos, mesAlvo);
  await carregarDados();
  _dupLock = false;
}

// DEPOIS:
async function iniciarNovaCompetenciaFu() {
  if (_dupLockFu) return;
  _dupLockFu = true;
  try {
    await carregarDados(); // Garantir dados frescos
    const mesAtualFu = mesSelecionadoFu || new Date().toISOString().slice(0,7);
    const [ano, mes] = mesAtualFu.split('-').map(Number);
    const dataSeguinte = new Date(ano, mes, 1); // JS months 0-indexed: mes=6 → July
    const mesAlvo = dataSeguinte.toISOString().slice(0,7);
    
    const origem = clientes.filter(c =>
      c.follow_up === true &&
      c.mes_referencia === mesAtualFu &&
      c.status !== 'emitido' &&
      c.convertido !== true &&
      c.convertido_fu !== true
    );
    
    if (origem.length === 0) {
      mostrarToast('Nenhum follow-up com F-U marcado neste mês.');
      _dupLockFu = false;
      return;
    }
    
    const existentesNoAlvo = clientes.filter(c => c.mes_referencia === mesAlvo).map(c => c.cliente);
    const todos = origem.filter(c => !existentesNoAlvo.includes(c.cliente));
    
    if (todos.length === 0) {
      mostrarToast('Todos os follow-ups já existem no mês seguinte.');
      _dupLockFu = false;
      return;
    }
    
    await duplicarClientesFu(todos, mesAlvo);
    await carregarDados();
    renderizarTabelaFu();
    atualizarStatsFu();
  } catch (e) {
    console.error('[NovaCompetenciaFu]', e);
    mostrarToast('Erro ao duplicar follow-ups');
  }
  _dupLockFu = false;
}
```

**Verificação:** Na aba FU do mês de Junho, marcar F-U em alguns clientes → clicar "+ Nova competência" → clientes aparecem na aba FU de Julho.

```javascript
async function converterClienteFu(id, checked) {

    const c = clientes.find(c => c.id === id);
```
