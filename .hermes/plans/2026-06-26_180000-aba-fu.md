## Task 8: Criar `converterClienteFu()` (checkbox Conversão na aba FU)

**Objetivo:** Checkbox "Conv." na aba FU migra o cliente para a aba Andamento (aba 2). Mesma lógica que `converterCliente()` da aba Análises.

```javascript
async function converterClienteFu(id, checked) {
  const c = clientes.find(c => c.id === id);
  if (!c) return;

  const mesRef = new Date().toISOString().slice(0,7);

  if (checked) {
    // Verificar se já existe clone na aba 2
    const jaExiste = clientes.some(x => x.cliente === c.cliente && x.mes_referencia === mesRef && x.convertido === true && x.id !== id);
    if (!jaExiste) {
      const { data: user } = await sb.auth.getUser();
      const clone = {
        indicacao: c.indicacao || '', cliente: c.cliente, cpf: c.cpf || '', agencia: c.agencia || '',
        valor_imovel: c.valor_imovel || '', valor_financiado: c.valor_financiado || '',
        produto: c.produto || 'SBPE', data_vistoria: '—', status: 'aguardando-doc', obs: '',
        created_by: user?.user?.id,
        criado_por: usuarioNome || user?.user?.email?.split('@')[0] || '—',
        convertido: true, mes_referencia: mesRef,
        fgts: c.fgts || false, iq: c.iq || false, banco: c.banco || '',
        banco_caixa: c.banco_caixa || false, banco_bradesco: c.banco_bradesco || false,
        banco_itau: c.banco_itau || false, banco_santander: c.banco_santander || false,
        banco_inter: c.banco_inter || false,
        valor_fgts: c.valor_fgts || '', data_iq: c.data_iq || '',
        data_matricula: c.data_matricula || '', urgente: false,
        data: c.data || '', analista: c.analista || ''
      };
      const { error } = await sb.from('clientes').insert(clone);
      if (error) { mostrarToast('Erro ao converter'); return; }
    }
    await atualizarCampo(id, 'convertido_fu', true);
    mostrarToast('Cliente convertido para Em Andamento ✅');
  } else {
    await atualizarCampo(id, 'convertido_fu', false);
    mostrarToast('Conversão desfeita');
  }

  await carregarDados();
  renderizarTabelaFu();
  renderizarTabelaAndamento();
  atualizarStatsFu();
}
```

**Verificação:** Marcar Conv. na aba FU → cliente aparece na aba Andamento com os mesmos dados.

## Task 11: Adicionar `iniciarNovaCompetenciaFu()` — clonagem mensal na aba FU

**Objetivo:** Na aba FU, clientes com F-U ✅ são clonados para o mês seguinte dentro da própria aba FU (mesma lógica antiga da aba Análises, agora aplicada à FU).

```javascript
async function iniciarNovaCompetenciaFu() {
  if (_dupLock) return;
  _dupLock = true;
  const [ano, mes] = (mesSelecionadoFu || new Date().toISOString().slice(0,7)).split('-').map(Number);
  const dataSeguinte = new Date(ano, mes, 1);
  const mesAlvo = dataSeguinte.toISOString().slice(0,7);

  const origem = clientes.filter(c => c.follow_up === true && c.mes_referencia === mesSelecionadoFu && c.status !== 'emitido' && c.convertido !== true && c.convertido_fu !== true);
  const existentesNoAlvo = clientes.filter(c => c.mes_referencia === mesAlvo).map(c => c.cliente);
  const todos = origem.filter(c => !existentesNoAlvo.includes(c.cliente));
  if (todos.length === 0) { mostrarToast('Nenhum follow-up para duplicar.'); _dupLock = false; return; }
  await duplicarClientesFu(todos, mesAlvo);
  await carregarDados();
  _dupLock = false;
}

async function duplicarClientesFu(lista, novoMes) {
  let count = 0;
  const { data: { user } } = await sb.auth.getUser();
  const userId = user?.id;
  for (const c of lista) {
    const { id, created_at, updated_at, ...dados } = c;
    dados.created_by = userId;
    dados.mes_referencia = novoMes;
    dados.follow_up = true; // manter F-U marcado no clone
    dados.status = dados.status || 'aguardando-doc';
    const { error } = await sb.from('clientes').insert(dados);
    if (error) { mostrarToast('Erro ao duplicar: ' + error.message); return; }
    count++;
  }
  mostrarToast(`📋 ${count} follow-ups duplicados para ${novoMes}`);
}
```

**Verificação:** Na aba FU, ao clicar em "Nova competência", clientes com F-U ✅ são clonados para o mês seguinte dentro da aba FU.

## Cuidados Específicos

### 1. Prevenção de dedup (cliente clonado 2x na FU)

- `atualizarCampo()` verifica se já existe clone com mesmo `cliente` + `mes_referencia` + `follow_up === true` antes de criar.
- Se já existe, mostra toast "Cliente já está no Follow Up deste mês" e não duplica.

### 2. Exclusão independente entre abas

- Cliente deletado na aba Análises: `excluirCliente()` faz soft-delete (`deleted_at`). O clone na FU é um registro separado com `id` diferente — NÃO é afetado.
- Cliente deletado na aba FU: `excluirCliente()` faz soft-delete do registro FU. O original na aba Análises NÃO é afetado.
- Ambos são registros independentes na tabela `clientes`, apenas compartilham o mesmo nome (`cliente`).

### 3. Sincronização de edições

- Edições nos campos compartilhados (cliente, CPF, valor, produto, status, bancos) **não são sincronizadas** entre o original (Análises) e o clone (FU). São registros independentes.
- Se o usuário editar "Cliente" na aba Análises, o clone na FU mantém o nome antigo. Para atualizar, editar diretamente na aba FU.
